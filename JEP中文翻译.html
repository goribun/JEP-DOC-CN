一、介绍

1.概述

JEP是一个用于解析和计算数学表达式的Java类库。通过使用这个包你可以把公式看作字符串并快速计算它们。其中内置了大量公共的数学函数和常量供用户使用。另外，你也可以通过自定义变量、常量、函数等方式扩展JEP。

1.1特性

· 占用空间小（281kb类库）

· 快速计算

·支持任意精度的算术

·支持字符串、向量和复数

·包括常用数学函数和预定义常量

·支持布尔表达式(!, &&, ||, <, >, !=, ==, >=, and <=)

·支持隐式乘法（允许使用“3x”代替“3*x”这种表达式）

·支持赋值表达式（例如x=5）

·允许选择声明变量或未声明变量

·可扩展的自定义函数

·可定制的语法表达式

·兼容J2SE 5

·支持字符编码（包括希腊符号）

·丰富的文档

·包括JavaCC语法

2.1解析

一般来说，可以描述解析作为一系列字符作为输入并产生一组包含字符集合信息的结构化表示。解析虽然用于其他领域例如自然语言解析等，但是这里的重点是数学表达式。

数学表达式可以通过树形结构表示，其中数字、变量、操作符和函数全部通过节点表示。数字和字符称为“子节点”，操作符和函数称为“父节点”。父节点操作它的子节点。例如，表达式“1+2*3”可以表示为一棵包含五个节点的树，如上图所示。

你也许会问为什么“乘”节点是“加”节点的一个子节点。这是运算优先级的原因。当解析表达式时，需要遵守优先级的顺序。乘法的优先级高于加法的优先级。所以，如上例所示，它需要在加1之前计算2乘以3。在树中，这是由乘法操作符的子节点数字2和3组成的。最后，乘法节点自己作为加法节点的一个子节点，计算乘法的结果用于计算加法操作。解析器按照优先级顺序负责确定语法。

解析器通常使用解析器生成器生成，例如JavaCC、ANTLR、SableCC、Yacc或者Bison。在众多解析器生成器中，JEP使用JavaCC。使用JccParser中定义的语法。JavaCC创建一组类来执行解析过程的核心任务。一个可配置的Java解析器也包括其中。

2.2计算

    计算把表达式从一个树形表示转变为一个表达式的值。从树通过操作符优先级建立，计算方法不需要知道操作符优先级。树可以使用一个简单的方法确定表达式的值来遍历。

一个简单的方法是从树的根节点（最顶层的节点）开始，运用递归方法。这个可以使用访问者模式来实现。不同的方法可以为每个节点类型创建。计算一个操作或者一个函数节点，首先子节点要被计算。然后，用子节点的值，操作符或者函数用于确定节点本身的值。常量（数字）或者变量节点，节点计算常量或者变量的值。使用这样的递归方法迅速地通过深度优先算法遍历树。

二、基本用法

1.入门指南

在你的项目中使用JEP非常简单。下面的步骤将帮助你快速开始。

1、下载JEP包

2、解压包

3、移动jep-java-x.x.x.jar文件(位于主目录)到所选择的目录中

4、重要：当编译程序时，确保Java编译器能够找到JEP 类，它需要知道它们的位置。你需要设置IDE中jar的位置来让编译器发现类库（如何去做请查看你的IDE帮助手册），如果你不是用IDE，你需要把jar加入到你的CLASSPATH环境变量中，例如 C:\java\packages\jep-java=x.x.x.jar，这取决于你把jar放到哪一路径。

5、将以下代码加入到你的程序中。

import com.singularsys.jep.Jep;

import com.singularsys.jep.JepException;

public class Demo {

public static void main(String[] args) {

Jep jep = new Jep();

try {

jep.addVariable("x", 10);

jep.parse("x+1");

Object result = jep.evaluate();

 

System.out.println("x + 1 = " + result);

} catch (JepException e) {

System.out.println("An error occurred: " + e.getMessage());

}

}

}

第五行创建了一个Jep解析器对象，第七行添加一个变量“x”，并为其赋初始值“10”，第八行和第九行，解析表达式“x+1”，并计算；计算结果保存在result中；最后，打印结果。

6、你应该能够编译运行你的应用，并看到打印值。

接下来是什么？通过浏览文档明白Jep都能做什么。也可以看下简单的示例applets。

2.错误处理

当错误发生，异常将被解析和计算方法抛出。解析方法抛出ParseException实例，计算方法抛出EvaluationException实例。两个异常类都继承自JepException类，所以如果你不需要区分解析错误和计算错误，你可以简单地抛出JepException实例。 

异常类通过getMessage()方法提供关于错误发生的信息。

3.默认设置

   如果你通过使用默认设置创建一个Jep实例，你可以解析带有以下函数列表中的函数的所以功能，你也可以使用以下常量：

pi

3.1415… 圆的周长比圆的直径（Math.PI）

e

2.718… 双精度欧拉常数（Math.E）

true

布尔常量（Boolean.TRUE）

false

布尔常量（Boolean.FALSE）

i

虚数单位

4.计算方法

三个用于计算表达式的方法：

·Object evaluate()：计算最后解析的表达式并作为对象返回结果。

·Object evaluate(Node root)：通过根节点和计算解析表达式并作为对象返回结果。

·double evaluateD()：计算最后解析的表达式并作为double类型返回结果，如果结果不能转换为double类型，将抛出EvaluationException异常。

   你可能总是不知道表达式结果类型。例如，它依赖表达式解析可能是Double，Vector，Boolean或者String。你可以用instanceof操作符去确定结果类型，然后把结果转换为相应的类。

5.快速重复计算

当在没有重复解析表达式而变量的值改变的时候执行重复计算表达式是可能的。当然，这比重复解析每次需要的计算要快的多。

下面的代码展示addVariable(String,Object)和evaluate()方法如何重复调用去改变变量的值以及重复计算表达式。

// add variable to allow parsing

jep.addVariable("x", 0);

jep.parse("2*x-3");

// evaluate expression for x = 0 to x = 99

for (int i = 0; i < 100; i++) {

// update the value of x

jep.addVariable("x", i);

// print the result

System.out.println("Value at x = " + i + ": " + jep.evaluate());

}

     你可以在任何计算器包括RealEvaluate()中使用快速重复计算。

6.Decimal运算

   在Jep 3中，使用Decimal运算计算表达式已经成为可能。这种数据类型在BigDecimal而不是double模式中表示数字。

   简单地使用下面的代码构建一个新的Jep实例。

jep = new Jep(new BigDecComponents());

这种不同的好处通过例子展示。当执行两个double类型相乘的操作，10*0.09计算结果为0.8999999999999999。但是当使用decimal运算（BigDecimal）执行同样的计算时，10*0.09的结果为0.9。

    如果你对高精度算法感兴趣，参考文档的BigDecimal部分。

7.隐式乘法

    你可以设置setImplicMul(true)使用隐式乘法操作。默认设置为true（允许隐式乘法）。

隐式乘法允许表达式“2 x”代替“2*x”。注意，两个变量之间的空格被看作是乘。这个方法适用于一个变量后跟着一个数字。例如“y 3”解释为“y*3”，但是“y3”解释为一个单独的变量“y3”。

如果一个变量在数字之前，它们之间不需要空格为隐式乘法。

8.处理多个表达式

   Jep可以同时处理多个表达式。简单的方法是在多个字符串中多次调用Jep.parse()方法。方法返回一个Node类型的对象表示表达式树。这些节点可以被存储用来以后使用或者使用Jep.evaluate(Node n)方法计算。

Jep jep = new Jep();

try {

Node n1 = jep.parse("y=x^2");

Node n2 = jep.parse("z=x+y");

for (double x = 0.0; x <= 1.0; x += 0.1) {

jep.addVariable("x", x);

Object value1 = jep.evaluate(n1);

Object value2 = jep.evaluate(n2);

}

} catch (JepException e) {

}

另一种方法允许所有的表达式包含在一个单独的字符串中。一系列等式可以使用Jep.initMultiParse(String s)方法从一个单独的字符串或者Reader之中，Jep.initMultiParse(Reader r)和continueParsing()允许所有的表达式包含在一个字符串之中。等式用分号隔开，解析器通过initMultiParse初始化，表达式通过使用continueParsing()轮流读。例如：

Jep jep = new Jep();

String s = "x=1; y=2; x+y";

jep.initMultiParse(s);

try {

Node n;

while ((n = jep.continueParsing()) != null) {

Object res = jep.evaluate(n);

}

} catch (JepException e) {

}

9.使用RealEvaluator快速计算

   从Jep 3.2到Jep 3.3，通过 FastEvaluator代替默认的StandardEvaluator组件使得计算速度有了重大提升。然而，如果你的表达式操作仅仅是单精度浮点数，你可以实现一个额外的RealEvaluator可以被加载通过Jep构造器Jep jep = new Jep(new RealEvaluator())；或者通过在构造器之后设置计算器jep.setComponent(new RealEvaluator())。

   性能提升依赖的因素包括：表达式，JVM和操作系统。在测试中，速度与FastEvaluator相比，大约提升30%。

三、变量

1.基础

     变量被Variable类表示并储存在VariableTable中。变量的值通过Jep.addVariable(String name,Object val)方法设置，通过Object Jep.getVariableValue(String name)方法检索。addVariable()方法也可以被用于改变已经存在的变量的值。对于double值，用 Jep.addVariable(String, double, double)方法。进一步的方法，getVariable(String)，返回Variable对象。这个类允许变量被观察，并提供了一个略微快速的方法getting和setting变量值。

2.未声明和未定义的变量

在解析期间，解析器查找是否存在变量。Jep.setAllowUndeclared(boolean flag)为未声明变量设置行为：

1、允许未声明变量 (默认)：当解析表达式时，未声明变量将被加入到Variable

2、不允许未声明变量：这种情况下，必须在在解析之前使用addVariable()方法加入变量。如果遇到未声明变量，将抛出ParseException 异常。如果你希望限制一个变量出现在表达式中，这个选项是很有用的。

在计算期间，变量的值将是可被检索的。如果值未定义（null），将抛出EvaluationException异常。如果没有使用 addVariable()设置变量值，null值将会出现。特别是在allowUndeclared设置为true，在解析期间创建变量。这个行为可以通过调用方法 StandardEvaluator.setTrapNullValues(boolean flag)来改变。设置默认值可以通过调用The Jep.setDefaultValue(Object)来实现。

3.赋值 

赋值允许变量的值通过使用“=”操作符设置，所以通过这种方式赋值是可能的：x = 3；在表达式中使用赋值例如：y = x^2；此时y的值为9。赋值是默认开启的。调用 setAllowAssignment(boolean)方法可以关闭赋值特性。为了Jep能够解析赋值等式，必须调用 setAllowUndeclared(true)方法。

重要：解析被赋值的变量后，必须调用 evaluate()方法。

// standard initialisation

Jep j = new Jep();

j.setAllowUndeclared(true);

 

// switch assignment facilities on

j.setAllowAssignment(true);

 

j.parse("x=3");

// evaluate it - no need to save the value returned

try {

j.evaluate();

} catch (EvaluationException e) {

}

 

j.parse("y=2");

try {

j.evaluate();

} catch (EvaluationException e) {

}

 

j.parse("x^y");

try {

Object val3 = j.evaluate();

System.out.println("Value is " + val3);

} catch (EvaluationException e) {

}

4.获取变量列表

如果你解析一个有未知变量（当允许未声明选项开启），你可能需要知道哪个变量在表达式中。有两种方式取得变量列表。

第一种是使用getVariableTable方法。通过返回的变量表你接下来可以在解析期间检查哪一变量已经被添加。特别是 VariableTable.getVariables()方法返回所有变量的list。VariableTable类中的toString()方法将提供一个可打印的list。

第二种方法是在解析表达式后使用 TreeAnalyzer类来检查表达式树。你可以查看JavaDoc文档来获取更多信息。

5.变量观察者

    Variable类实现了观察者模式。当变量值改变时允许对象被通知。使用 Variable.addObserver(Observer)方法注册一个类成为变量的观察者。VariableTable还可以被观察去检查新变量。 VariableTableObserver使得观察VariableTable和它的变量变得容易。

 

四、数据类型

Jep在标准模式中支持很多数据类型。包括Number，String，Vectors和复数。默认类型为double类型。然而改变内部表示是可能的就像自定义类型。

1.String

通过使用双引号可以将字符串输入到表达式中。它们可以通过“+”连接，可以通过“==”和“!=”比较。例如 "foo" + "bar" == "foobar"，在Jep中计算为true。

使用 addVariable(String name, Object value)方法加入字符串变量。如果表达式结果是一个字符串，它可以通过调用evaluate()方法获得。

2.Vector

   Vector是一系列值的列表，使用逗号分开，使用方括号封闭。你可以使用方括号访问一个元素。例如，使用v[2]来访问v的第二个元素的值。

以下列出示例包含Vector的表达式。

a = [2, 3, 10]   // create a variable with a vector value

a[1]             // evaluates as 2

a[3]             // evaluates as 10

 

a + [1, 2, 3]    // addition, evaluates as [3, 5, 13]

a - [1, 2, 3]    // subtraction, evaluates as [1, 1, 7]

[3, 4, 5]*2      // multiplication by a scalar,

                 // evaluates as [6, 8, 10]

[1,2] * [1,2]    // interpreted as a column vector times a row

                 // vector. Evaluates as [[1.0, 2.0], [2.0, 4.0]]

[4, 6, 8]/2      // division by a scalar, evaluates as [2, 3, 4]

[1,2,3].[1,2,3]  // dot product, evaluates as 14

[1,2,3]^^[1,2,3] // cross product, evaluates as [0, 0, 0]

min([6,4,7,8])   // minimum value, 4

max([6,4,7,8])   // minimum value, 8

avg([6,4,7,8])   // average value, 2.5

vsum([6,4,7,8])  // sum of elements, 25

在内部，vector的值表示为list，尤其是Vector<Object>。使用 addVariable(String name, Object value)将一个vector作为变量加入。值的参数应该是Vector<Object>类型。如果表达式结果是一个vector，它可以通过调用evaluate()方法获得。

Vector变量可以通过以下方式使用：

 // Add the variable y with value [1.2,3.4]

    jep.addVariable("y", new Vector<Object>(Arrays.asList(new Double[]{1.2,3.4})));

    // Use the y[2] syntax to get a specific element

    Object res = jep.evaluate(jep.parse("y[2]"));

    assertEquals(3.4,res);

    // calculate the vector expression y*2 

    Object res2 = jep.evaluate(jep.parse("y*2"));

    // The returned type will be Vector<Object>

    assertTrue(res2 instanceof Vector<?>);

    Vector<Object> vec = (Vector<Object>) res2;

    // Convert to an array

    Object[] array = vec.toArray();

assertArrayEquals(new Object[]{2.4,6.8},array);

 

   非常有限的支持矩阵。矩阵可以被嵌套的vector [[1.0, 2.0], [2.0, 4.0]]。Vector乘以Vector将得到一个矩阵。但是矩阵乘以Vector或者矩阵乘以矩阵不能得到合理的结果。Vector可以使用min,max,avg和vsum函数，折叠所有元素至一个一维数组。

3.复数

Jep在大多数操作符和函数中支持复数，复数的虚数单位i是变量表标准的一部分，当使用StandardComponents时，它会被加入到解析器。一组复数函数也被包含其中。包括re(c)，im(c)，cmod(c)，arg(c)，conj(c)，complex(x,y)和polar(r,theta)。查看更多信息请查看functions page，你需要导入 com.singularsys.jep.standard.Complex来使用它。

通过使用虚数单位i，你可以在表达式中使用复数，下面为例子：

(1+2*i)^3

e^(-i)

re((1+2*i)^2 - (3+3*i)^2)

   从表达式中取得复数的值，你必须使用evaluate()方法。它将计算表达式并返回Complex类型结果。

   你需要手动地将Object类型转换为Complex类型。

在计算表达式之前，往解析器中添加复数类型的变量或者常量，必须通过addVariable(String name, double re, double im)方法。它包括三个参数，字符串类型的变量名，实部和虚部。

4.自定义类型

   大部分情况下，你只需要建立少量的自定义类型，因为Jep已经支持大量数据类型。但是你想要计算包括其他类型的表达式。你可能需要让变量拥有自定义类型或者创建自定义类型数字。

你可以使用addVariable(String name, Object value) 方法创建自定义类型变量。

你可以确定能够通过设置number工厂创建特殊类型的number。例如BigDecimal组件可以创建比double类型更加精确的数字。简单地实现NumberFactory接口和 createNumber(String) 方法，调用setComponent(JepComponent)载入你的number工厂。

当一个表达式被计算，值被功能包内的class操作，包括操作符（例如加和减），还有函数（例如Sine和Cosine）。不修改源代码，只能操作默认类型。所以，为了能够操作自定义类型，你需要修改类，或者让你的函数类像自定义函数那样。更多信息请参考 custom functions 部分。

2.加入操作符

   提供的解析器允许加入新的操作符。configurable parser让这些变得简单，但是需要使用默认的JavaCC解析器。新操作符用过通过Jep实例加入到OperatorTable中。

在试图加入新的操作符之前，你应该熟悉加入一个 custom function到Jep。一个 PostfixMathCommandI (pfmc)实例被用于控制操作符的计算。

3.为可配置的解析器加入和操作操作符

往可配置的解析器中加入操作符，它们首先必须被加入到Operator表，然后调用jep.reinitializeComponents()方法通知解析器。一旦操作，解析器将能够使用新操作符。在被调用之前操作符的优先级和关联性的设置是非常重要的。

3.1修改已存在的操作符

   如果你只希望改变PostfixMathCommandI底层操作符，例如允许它操作新的值类型，保持存在的优先级和关联性以及符号，然后Operator的setPFMC()方法被调用。操作符可以通过OperatorTable的get方法中恢复。例如改变PostfixMathCommandI和”(& &)操作符的使用：

jep.getOperatorTable().getAnd().setPFMC(new LazyLogical(LazyLogical.AND));

如果你希望改变操作符的符号你可以使用：

jep.getOperatorTable().getAnd().setSymbol("AND");

jep.reinitializeComponents();

注意调用jep.reinitializeComponents()通知新符号的其他组件。

3.2优先级和绑定

符号可以是二进制加操作符(a+b)，一元前缀操作符(!a)，一元后缀操作符(i.e 像Java的x++尽管没有后缀定义为标准)或者三元操作符例如Java的a?b:c，也没定义为标准。静态字段为每个Operator类中的操作符定义标记。它们包括 Operator.BINARY， Operator.UNARY，Operator.PREFIX， Operator.SUFFIX和 Operator.NARY为了带有3个及以上参数的操作符。

操作符解析的顺序取决于他们的优先级。例如1+2*3为1+(2*3)。“*”操作符优先级高于“+”操作符。在Jep中我们使用这样的约定： that low values for precedence give a tighter binding, so the numeric value of the precedence of * is less than that of +. Precedences are always set in a relative manner as discussed below。

操作符的解析也依赖于绑定或者结合性。可能是左结合（使用 Operator.LEFT），像=where a=b=c被解释为a=(b=c),或者右结合（Operator.RIGHT），像- where a-b-c被表示为(a-b)-c。向左或者向右应该为所有二元操作符指定。Operator.ASSOCIATIVE被用于表名关联操作符像“+”用于打印所程序来确定是否包括括号。

构造一个操作符构造函数 public Operator(String name,PostfixMathCommandI pfmc,int flags)用于给符号用于操作符， PostfixMathCommandI和标记总和。第二个构造函数public Operator(String name,String symbol,PostfixMathCommandI pfmc,int flags)用于两个操作符使用同一个符号的情况，例如：

Operator add = new Operator("+",new Add(),

Operator.BINARY+Operator.LEFT+Operator.ASSOCIATIVE);

Operator sub = new Operator("-",new Subtract(),

Operator.BINARY+Operator.LEFT);

Operator mul = new Operator("*",new Multiply(),

Operator.BINARY+Operator.LEFT+Operator.ASSOCIATIVE);

Operator div = new Operator("/",new Add(),

Operator.BINARY+Operator.LEFT);

Operator umin = new Operator("UMinus","-",new UMinus(),

Operator.UNARY+Operator.RIGHT+Operator.PREFIX);

Operator equals = new Operator("=",new Assign(),

Operator.BINARY+Operator.RIGHT);

 

     有一些将来可用的标记，指定其他目前不可用的操作符的数学属性。使用com.singularsys.jep.configurableparser.TernaryOperator类用于三元操作符。

3.3将操作符加入操作符表

   从3.3版本开始，已经引进了新的 com.singularsys.jep.OperatorTable2类。每种类型给定一个key。例如OperatorTable2.BasicOperators.ADD是一个用于加操作的key。使用replaceOperator(OperatorKey key,Operator op)可以替换操作符，使用removeOperator(OperatorKey key)可以删除操作符。使用Operator getOperator(OperatorKey key)可以从操作符表中恢复操作符。给一个新操作符一个新的key必须实现 OperatorTable2.OperatorKey，并使用以下方法加入表中：

 

   · Operator addOperator(OperatorKey key, Operator op)

   ·public Operator addOperator(Operator op, Operator existingOp)

   ·public Operator insertOperator(Operator op, Operator existingOp)

   ·public Operator appendOperator(Operator op, Operator existingOp)

第一个添加未设置优先级的操作符，第二不加入一个和现有操作符一样优先级的操作符，第三个和第四个创建一个新的优先级，相对于已存在的操作符在(i.e. tighter)之前或在(i.e. looser)之后。

// Set up the keys

...

// Build the operator

                       new Factorial(),

                       Operator.SUFFIX+Operator.UNARY+Operator.LEFT);

                           new Percent(),

                           Operator.SUFFIX+Operator.UNARY+Operator.LEFT);

                           

// Add operators with same precedence as the POSFIX operator                           

addOperator(FACT, factop,  getOperator(POSTFIX));

addOperator(PERCENT, pcop, getOperator(POSTFIX));

4.使用标准解析器添加操作符

   将操作符添加到标准解析器中目前唯一方式是修改Jep源代码。为了将操作符添加到解析器中，需要几个步骤：

  ·创建一个新的令牌描述名AT，say和一个字符串代表操作符。

  ·修改语法规则，包括新的操作符的规则。

  ·添加一个操作符正如上面所讨论的。

4.1编辑标准的解析器语法

标准的解析器是使用JavaCC解析器/生成器创建的。JavaCC读取使用专用语言编写的JccParser.jjt文件，定义语法。输出 JccParser.java文件，其他一些实现的Java文件。

在修改解析器之前，你应该读一些关于JavaCC和JJTree的文档。

以下三个步骤用于生成解析器。

1.JJTree是运行的，它读JccParser.jjt文件，创建 JccParser.jj文件。这步的目的是添加必要的代码来处理抽象语法树（树结构应用代表一个数学表达式）。像创建JccParser.jj 一样，它也会创建一些其他Java文件:Node.java（在解析器中表示基础节点）；ASTConstant.java（表示一个常量值“0”或“1”的节点）；ASTVarNode.java（表示一个变量的节点）；ASTFunNode.java（表示函数或操作符的节点）。

2.JavaCC运行。读取JccParser.jj并创建 JccParser.java--实现Parser的实际代码。JccParser.java 是一个非常复杂的文件有近2000行代码而且编辑非常困难。

3.普通的Java编译JccParser.java和其他类通过JavaCC产生。

在项目使用ANT build.xml文件时这个过程是自动的。仅仅重编译所以Java文件是不够的。

只有JccParser.jjt文件应该被修改，JccParser.jj和JccParser.java在构建期间重写不应该被修改。此外，ASTConstant.java， ASTFunNode.java，ASTStart.java， ASTVarNode.java，JavaCharStream.java， JJTParserState.java，Node.java, SimpleNode.java， ParseException.java，JccParserConstants.java， JccParserTokenManager.java， JccParseTreeConstants.java， Token.java和 TokenMgrError.java在自动生成期间也不应该被修改。

七、自定义函数

Jep允许你通过调用getFunctionTable().addFunction(String name, PostfixMathCommandI function)方法添加自定义函数。自定义函数和内建函数没有什么不同，所以你可以使用内建函数源码作为模版。整个文档习惯使用缩写pfmc引用这些类。

为了创建一个自定义函数，你必须首先创建一个继承以下一个父类的新类：

UnaryFunction：为一个参数函数；Object eval(Object l)必须被实现。

BinaryFunction：为两个参数函数；定义Object eval(Object l,Object r)方法。

NaryFunction：为任意个数参数函数；定义Object eval(Object[] args)方法。

PostfixMathCommand：为任意参数参数函数；使用一个栈传递参数通过run(Stack<Object> stack)方法。

UnaryFunction，BinaryFunction和NaryFunction是Jep 3.3中全新的，更容易实现并且更快计算。

1.UnaryFunction和BinaryFunction

这些是最简单的实现。一个简单的一个一元平方函数的实现类：

import com.singularsys.jep.EvaluationException;

import com.singularsys.jep.functions.UnaryFunction;

 

public class Square extends UnaryFunction {

private static final long serialVersionUID = 1L;

@Override

public Object eval(Object l)  {

double x = ((Number) l).doubleValue();

return x * x;

}

}

 

    加入到Jep中使用：

jep.addFunction("square", new Square());

 

   二元函数和以上类似，但是必须实现eval(Object l,Object r)方法。 

2.参数类型转换和异常

以上的square()方法能够正常工作，但是它不能很好的处理非法值，如果传入一个字符串它将抛出ClassCastException异常。如果EvaluationException或子类 IllegalParameterException抛出，改进的表现是必要的。前者是用于一般错误，后者是特定非法参数标准功能和生成一个错误消息。升级版如下：

public Object eval(Object l) throws EvaluationException {

// if incorrect type throw an exception

// specifying position in argument list, expected type and supplied

// value.

if (!(l instanceof Number))

throw new IllegalParameterException(this, 0, Number.class, l);

double x = ((Number) l).doubleValue();

return x * x; // result is auto-boxed to return a Double.

}

PostfixMathCommand和它的子类为转变参数提供了大量的便捷方法： asInt, asStrictInt, asLong, asDouble, asString, asBool,它们在必要的时候返回相应的原始类型、抛出异常。用asDouble方法实现的代码如下：


 public Object eval(Object l) throws EvaluationException {

// Convert argument at position 0 to double, throwing exception if

// needed

double x = asDouble(0, l);

return x * x;

}

3.Nary functions

     如果函数接收0，大于2或者一个变量有几个参数，那么可以使用NaryFunction类。它的传入参数是一个对象数组Object eval(Object[] args)。默认情况下，函数接收任意个数的参数。在构造函数public NaryFunction(int nArgs)中可以指定固定的函数个数。参数中的变量个数可以通过重写public boolean checkNumberOfParameters(int n)方法指定，如果n是一个有效的参数变量，则返回true。在解析和计算期间这两种方法将检查参数个数。

例如下面的函数在0变量时总是返回当前时间。

import com.singularsys.jep.functions.NaryFunction;

public class Time extends NaryFunction {

private static final long serialVersionUID = 1L;

public Time() {

super(0); // does not allow any arguments

}

public Object eval(Object[] args) {

return System.currentTimeMillis();

}

}

下面的平方和计算允许两个或三个参数。

import com.singularsys.jep.EvaluationException;

import com.singularsys.jep.functions.NaryFunction;

public class SumSq extends NaryFunction {

private static final long serialVersionUID = 1L;

// Use default constructor: variable number of arguments

@Override

public boolean checkNumberOfParameters(int n) {

return (n == 2 || n == 3);

}

public Object eval(Object[] args) throws EvaluationException {

double x = asDouble(0, args[0]);

double y = asDouble(1, args[1]);

if (args.length == 2)

return x * x + y * y;

else {

double z = asDouble(2, args[2]);

return x * x + y * y + z * z;

}

}

}

 

更一般的版本将允许一个或多个参数：

import com.singularsys.jep.EvaluationException;

import com.singularsys.jep.functions.NaryFunction;

public class SumSq2 extends NaryFunction {

private static final long serialVersionUID = 1L;

// Use default constructor: variable number of arguments

// Ensure at least one argument

@Override

public boolean checkNumberOfParameters(int n) {

return (n == 2 || n == 3);

}

public Object eval(Object[] args) throws EvaluationException {

double x = asDouble(0, args[0]);

double res = x * x;

for (int i = 1; i < args.length; ++i) {

double y = asDouble(i, args[i]);

res += y * y;

}

return res;

}

}

4.基于PostfixMathCommand的栈

这个古老类型PostfixMathCommand仍然是可用的。这些是使用基于通过 run(Stack<Object> inStack)方法计算的栈。

1.创建一个继承自 PostfixMathCommand的类（在com.singularsys.jep.functions包）。在这个例子中我们把它命名为“Half”。

2.在构造方法中设置参数的个数，在我们的例子中是一个。通过编写numberOfParameters = 1实现。如果你想允许更多参数个数，初始化numberOfParameters value to -1，强烈建议将Sun.java作为接收多个参数个数的函数的例子来学习。

3.实现run(Stack<Object> inStack)方法。通过栈（Stack）传递参数。同样栈也用于函数的输出。所以首先我们需要弹出栈顶元素（pop），计算结果，最后将计算结果压入栈中（push）。注：原文为：So we first need to pop the parameter off the stack, calculate the result, and finally pop the result back on the stack.  感觉是书写错误！

@Override

public void run(Stack<Object> inStack) throws EvaluationException {

// check the stack

checkStack(inStack);

// get the parameter from the stack

Object param = inStack.pop();

 

// check whether the argument is of the right type

if (param instanceof Double) {

// calculate the result

double r = ((Double) param).doubleValue() / 2;

// push the result on the inStack

inStack.push(new Double(r));

} else {

throw new EvaluationException("Invalid parameter type");

}

}

4.在你的主程序或者applet中，使用解析器，加入新的函数，按照以下代码编写：

jep.getFunctionTable().addFunction("half", new Half());

5.如果numberOfParameters == -1，在解析期间，你可能希望重写boolean checkNumberOfParameters(int n)方法来捕获参数的非法数字异常。

6.在Jep版本3.3中引入了新的 IllegalParameterException异常。这是常见的参数类型错误。另外，PostfixMathCommand提供了String asString()， int asInt()， int asStrictInt()， int asLong()， int asDouble()方法让参数转换为需要类型变得简单。如果参数不是需要的类型，将抛出IllegalParameterException异常。上面的run方法可以简化为：

public void run(Stack<Object> inStack) throws EvaluationException {

checkStack(inStack);

// get the parameter from the stack

Object param = inStack.pop();

// convert param to a double throwing an exception

// the first argument is the name of the function, and the second is its

// position

double val = asDouble("half", 1, param);

 

double r = val / 2;

inStack.push(new Double(r));

}

5.特殊函数

其他还可以使用基类和接口:

NaryBinaryFunction：函数类似于加和乘，可以使用多个参数，但基础实现是一个二元函数。只需要实现 Object eval(Object l,Object r)方法。

CallbackEvaluationI：传递的参数不是值而是树节点，通过调用函数独立的子节点被计算。当不是所有的分支都被计算时它允许懒计算。它也允许变量名被发现。

LValueI：被用于左边赋值的函数的接口。

八、BigDecimal模式

   在Jep3中，使用十进制运算（decimal arithmetic）来计算表达式成为可能。它允许指定计算精确度，甚至无限的精确度。如果你遇到精确度问题或者已经知道你需要更高的精确度，我们建议使用这种操作模式。

使用BigDecimal 表示数字，而不是使用Double类型表示数字。BigDecimal类是Java API的一部分，在java.math包下。想获得更多的相关信息，请阅读SUN Javadocs for BigDecimal。

通过一个例子展示这种不同。当执行两个Double类型数字相乘时：

10*0.09 evaluates as 0.8999999999999999。

但是当使用BigDecimal类型执行同样的计算：

  10*0.09 evaluates as 0.9。

1.如何使用BigDecimal模式

使用Jep的BigDecimal模式非常容易。使用BigDecComponents创建Jep实例：

jep = new Jep(new BigDecComponents());

这是通过BigDecNumberFactory，BigDecOperatorTable 和BigDecFunctionTable实例化Jep。默认情况下，数学环境设置无限的精确度。但是你也可以使用不同的数学环境实例化组件。例如，使用32位精确度：

jep = new Jep(new BigDecComponents(MathContext.DECIMAL32));

获取更多有关MathContexts信息请参考Sun MathContext Javadocs。

2.支持的操作符和函数

不幸的是保证精确度是以比基本操作符和函数更复杂的算法为代价。由于这个原因，一些函数和操作符在double计算中可以使用，而在BigDecimal中不可使用。

以下操作符和函数是支持的：

   +, -, *, /, ^ (幂), % (模)

   >, <, >=, <=, == (equals)

   && (与), || (或), ! (非)

   像在Jep标准模式一样，你可以增加自定义函数。

3.BigDecimal和字符串

在3.4版本之前，默认情况下BigDecimal和字符串是不能一起使用的。下面的代码打开了这一特性，并增加了标准字符串函数。

// Create the BigDecComponents

BigDecComponents compSet = new BigDecComponents(MathContext.DECIMAL64,

true);

// Create a jep instance

jep = new Jep(compSet);

// Add the standard set of string functions

jep.setComponent(new StringFunctionSet());

 

      以上代码允许字符串使用“+”连接，使用“==”，“!=”，“<”，“>”，“<=”，“>=”，“>”比较。StringFunctionSet 允许使用left，right，lower，upper，subsring，len，mid，trim。

九、常见问答

1.为什么Jep给定的结果不准确？

你注意到当你计算例如“8-7.9”时，结果是0.09999999999999964而不是0.1。这些错误结果是由于浮点运算导致的。在内部，Jep默认使用double类型表示数字。不幸的是，甚至连简单的“8-7.9”都不能准确的执行。

你注意到允许以下代码的同样问题：

double a = 8, b = 7.9;

System.out.println("a-b = " + (a - b));

// prints a-b = 0.09999999999999964}

所以，这不是Jep的错误，只是使用浮点数的限制。虽然浮点数对于大多数应用是足够精确的，但是在精确性特别重要的应用中这些问题是不能忽略的。

使用system . out.printf的结果中可以显示给定数量的小数。例如：

 jep.parse("8-7.9");

double res = jep.evaluateD();

System.out.printf("%.3f", res);

这将打印结果到小数点后三位:0.100。查看java.util.Formatter获取格式化输出详细信息。java.text.NumberFormat类也可以用于结果格式化。

Jep有取整函数，可以参数给定数量的小数位。round(8-7.91)结果将为一个小数位。

2.我如何处理NULL值？

   如果Jep在计算期间抛出“EvaluationException: Could not evaluate XXX : no value set for the variable”异常，那么运行中存在null问题。

   默认情况下，变量的值既没有通过jep.addVariable(name,value)设置，又没有通过表达式解析：XXX = 123.456;去初始化null是要特别注意的。所以当你打算计算和解析一个null变量，计算器默认会在遇到null值时抛出一个异常。

有以下几点：

1.依赖你使用的计算器，你可以关闭null值检查：((StandardEvaluator)jep.getEvaluator()).setTrapNullValues(false);

或者

((FastEvaluator)jep.getEvaluator()).setTrapNullValues(false);

这种情况下，空值将被传到函数中。

2.你可以改变未赋值的变量的默认值：

jep.getVariableFactory().setDefaultValue(Double.valueOf(0.0));

3.对于null值的最高级的处理方式是创建一个类型来表示真实的null值，创建一个常量NULL来识别解析器的NULL，明确地设定NULL变量。

Object MyNULL = new Object(); // create a null value

jep.addConstant("NULL", MyNULL); // define a constant in the parser

jep.addVariable("x", MyNULL); // add a variable with the NULL value

jep.parse("y=NULL"); // expression assigning a variable to NULL

jep.parse("x==NULL"); // Compare a variable to the NULL value

4.从3.4版本开始，src.com.singularsys.jep.misc.nullwrapper包已经允许null值就像数据库中的null值那样。任何函数或操作符以null作为参数将返回null。例如null+5将返回null；包使现存的操作符、函数和添加一个包装类来测试是否输入了null值，如果是，则返回null。否则使用原来的函数。NullWrappingOperatorTable和NullWrapperFunctionTable包含所有的已存在的操作符和函数。以上设置使用：

jep.setComponent(new NullWrappingOperatorTable((OperatorTable2)         jep.getOperatorTable(), true));

jep.setComponent(newNullWrapperFunctionTable(jep.getFunctionTable()));

jep.setComponent(new StandardConfigurableParser());

((FastEvaluator) jep.getEvaluator()).setTrapNullValues(false);

操作符表也可以添加额外的操作符，null安全等于<=>，StandardConfigurableParser加入它至解析器。

3.为什么我得到一个EvaluationException

很多原因会导致抛出EvaluationExceptions异常，当然最好的做法是仔细检查异常的详细信息。

如果你看到：

com.singularsys.jep.EvaluationException: Could not evaluate XXX : no

    value set for the variable.

   这参考“常见问答”中的“我怎么处理null值”部分。

   其他计算异常通常在函数或操作符传入了不兼容的数据类型。只在计算期间，Jep才会检查数据类型，在解析表达式期间你不会看到任何问题。

4.我如何允许带点的变量名？

做到这个，你需要使用允许控制输入语法的Configurable Parser。下面的代码设置了解析器的标准选项，但是使用符号“.”还需要使用工厂方法com.singularsys.jep.configurableparser.matchers.IdentifierTokenMatcher.dottedIndetifierMatcher()。最好的步骤是设置是删除dot-product "." 操作符，以防止歧义。

ConfigurableParser cp = new ConfigurableParser();

cp.addHashComments();

cp.addSlashComments();

cp.addDoubleQuoteStrings();

cp.addWhiteSpace();

cp.addExponentNumbers();

cp.addOperatorTokenMatcher();

cp.addSymbols("(", ")", "[", "]", ",");

cp.setImplicitMultiplicationSymbols("(", "[");

 

// Sets it up for identifiers with dots in them.

cp.addTokenMatcher(IdentifierTokenMatcher.dottedIndetifierMatcher());

 

cp.addSemiColonTerminator();

cp.addWhiteSpaceCommentFilter();

cp.addBracketMatcher("(", ")");

cp.addFunctionMatcher("(", ")", ",");

cp.addListMatcher("[", "]", ",");

cp.addArrayAccessMatcher("[", "]");

 

// Construct the Jep instance and set the parser

jep = new Jep(cp);

 

// Remove the dot operator

OperatorTable2 ot = (OperatorTable2) jep.getOperatorTable();

ot.removeOperator(ot.getDot());

// notify other components of change in operator table

jep.reinitializeComponents();
